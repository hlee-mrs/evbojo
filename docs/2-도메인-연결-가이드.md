## 2. 연결 구조 (권장: Cloudflare 무료 프록시)
```
방문자 → Cloudflare(HTTPS·캐싱·IP숨김) → 집 공유기(80/443 포워딩) → NAS(8080)
```
집 인터넷은 IP가 바뀌므로(유동 IP) 아래 중 하나로 대응합니다.

### 방법 A — Cloudflare + DDNS 스크립트 (권장)
1. cloudflare.com 가입 → 사이트 추가 → 도메인 네임서버를 Cloudflare로 변경(등록기관 설정에서)
2. DNS 레코드: `A` `@` → 현재 집 공인 IP, 프록시(주황 구름) **ON**
3. 유동 IP 자동 반영: Synology 작업 스케줄러에 10분마다 아래 실행
   (Cloudflare API 토큰 발급: My Profile → API Tokens → Edit zone DNS)
```bash
#!/bin/bash
TOKEN="발급한_토큰"; ZONE_ID="존ID"; REC_ID="레코드ID"; DOMAIN="evbojo.com"
IP=$(curl -s https://ifconfig.me)
curl -s -X PUT "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/dns_records/$REC_ID" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  --data "{\"type\":\"A\",\"name\":\"$DOMAIN\",\"content\":\"$IP\",\"proxied\":true}"
```
4. Cloudflare SSL/TLS 모드: "Flexible"(NAS에 인증서 없을 때) 또는 "Full"(Synology LE 인증서 있을 때)
5. 캐싱 규칙(선택): `/data/*` Cache TTL 10분 — NAS가 꺼져도 잠시 서빙됨

### 방법 B — Synology DDNS + 도메인 CNAME
1. 제어판 → 외부 액세스 → DDNS → `xxx.synology.me` 등록 (무료)
2. 도메인 DNS에서 `CNAME` `www` → `xxx.synology.me`
   (루트 도메인 @는 CNAME 불가인 등록기관이 많음 → ALIAS/ANAME 지원 확인 또는 Cloudflare 사용)

### 방법 C — 도메인 없이 시작
`xxx.synology.me`로 먼저 운영 시작도 가능해요. 단, **애드센스는 커스텀 도메인을 권장**(서브도메인 승인 불리)하므로 수익화 전에는 도메인을 붙이세요.
